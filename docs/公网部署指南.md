## 方案 A：Cloudflare Pages（推荐，免费 + 可绑域名）

1. 注册 [Cloudflare](https://dash.cloudflare.com/sign-up)（免费）
2. 左侧 **Workers & Pages** → **Create** → **Pages** → **Connect to Git**
3. 授权 GitHub，选择仓库 **`Daisyqiuj/EU-investment-screening-China`**
4. 构建设置：
   - **Framework preset**: None
   - **Build command**: （留空）
   - **Build output directory**: `/`（根目录）
5. 点击 **Save and Deploy**，约 1–2 分钟
6. 获得免费网址：`https://eu-investment-screening-china.pages.dev`（名称可改）

**绑定自己的域名（可选）**  
Pages 项目 → **Custom domains** → **Set up a custom domain** → 输入 `www.您的域名.com`，按页面提示在 Cloudflare 或原域名商添加 DNS。

## 方案 B：Netlify（同样免费）

1. 注册 [Netlify](https://app.netlify.com/signup)
2. **Add new site** → **Import an existing project** → **GitHub**
3. 选择仓库 **`EU-investment-screening-China`**
4. 构建设置（仓库已含 `netlify.toml`，一般自动识别）：
   - Publish directory: `.`（根目录）
5. **Deploy site**
6. 免费网址形如：`https://随机名.netlify.app`  
   可在 **Domain settings** 改为 `eu-fdi-screening.netlify.app` 等

**绑定自己的域名**  
**Domain settings** → **Add custom domain** → 按提示在域名商添加 CNAME 到 Netlify。

---

## 方案 C：继续用 GitHub Pages + 自定义域名

若您**已购买域名**（阿里云、腾讯云、GoDaddy、Cloudflare 等）：

1. 在仓库根目录创建文件 **`CNAME`**（仅一行，不要 `https://`）：
   ```
   www.您的域名.com
   ```
2. 提交并 push 到 GitHub
3. 仓库 **Settings → Pages → Custom domain** 填同一域名并保存
4. 在域名商 DNS 添加：
   - **CNAME** `www` → `daisyqiuj.github.io`
   - 或 **A 记录** `@` → GitHub Pages IP（见 [GitHub 文档](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site)）
5. 等待 DNS 生效（几分钟到 48 小时），开启 **Enforce HTTPS**

访问：`https://www.您的域名.com/web/index.html`  
可在域名商设置根域名跳转到 `www`。
